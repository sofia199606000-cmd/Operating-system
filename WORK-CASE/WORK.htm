Для чого використовують git?

 

Git — це найпопулярніша у світі система контролю версій, яка використовується для відстеження змін у файлах та спільної роботи над програмними проєктами. Вона дозволяє розробникам зберігати історію кожного виправлення, легко повертатися до попередніх станів коду, створювати окремі гілки для нових функцій без ризику зламати основний продукт, а також синхронізувати роботу між багатьма учасниками команди. Основна суть Git полягає у створенні «знімків» (commits) вашого проєкту, що робить процес розробки прозорим та безпечним.

Основними діями в Git є ініціалізація сховища, фіксація змін та обмін даними з віддаленим сервером (наприклад, GitHub або GitLab).

Для цього використовують такі базові команди:

 **git init** — створення нового локального репозиторію в поточній папці.

 **git clone** [url] — копіювання існуючого проєкту з віддаленого сервера на ваш комп'ютер.

 **git status** — перегляд стану робочої директорії (які файли змінені, а які готові до збереження).

 **git add** [file] — додавання файлів до області підготовки (staging area) перед фіксацією.

 **git commit -m "опис"** — створення «знімка» змін із пояснювальним коментарем.

 **git branch [name]** — створення нової гілки для розробки окремої функції.

 **git checkout [name] або git switch [name]** — перемикання між різними гілками.

 **git merge [name]** — об'єднання змін з однієї гілки в іншу (зазвичай у головну — main).

 **git push** — відправка ваших локальних комітів на віддалений сервер.

 **git pull** — завантаження та об'єднання останніх змін від колег із сервера у ваш локальний код.
