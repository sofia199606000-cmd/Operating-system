### Словник основних англійських термінів (Віртуальні середовища)

**Virtual Machine** — програмна емуляція фізичного комп'ютера, на якому працює операційна система та додатки.

**Virtualization** — технологія, що дозволяє запускати кілька операційних систем на одному фізичному комп'ютері.

**Hypervisor** — програмне забезпечення або вбудоване програмне забезпечення, що створює та управляє віртуальними машинами.

**Type 1 Hypervisor** — гіпервізор, що працює безпосередньо на фізичному обладнанні без операційної системи хоста.

**Type 2 Hypervisor** — гіпервізор, що працює поверх операційної системи хоста.

**Host Operating System** — основна операційна система, встановлена на фізичній машині.

**Guest Operating System** — операційна система, встановлена та запущена всередині віртуальної машини.

**Kernel** — основний компонент операційної системи, що управляє апаратними ресурсами.

 ---

### Поняття та типи гіпервізора

Гіпервізор — це програмне забезпечення, яке використовується для створення та управління віртуальними машинами. Воно дозволяє одночасно запускати кілька операційних систем на одному фізичному комп'ютері та розподіляє між ними апаратні ресурси.

Згідно з теоретичним матеріалом, існує два основних типи гіпервізорів.  
Гіпервізори типу 1 працюють безпосередньо на апаратному забезпеченні без операційної системи хоста і зазвичай використовуються на серверах.  
Гіпервізори типу 2 встановлюються поверх існуючої операційної системи і частіше використовуються на персональних комп'ютерах.

### Основні компоненти та можливості ( Варіант 5 - Hyper-V )

Hyper-V є гіпервізором типу 1 і працює безпосередньо на апаратному забезпеченні комп'ютера без використання іншої операційної системи хоста. Він дозволяє користувачам створювати та керувати віртуальними машинами з різними гостьовими операційними системами.

Основними компонентами Hyper-V є гіпервізор Hyper-V, батьківський розділ (операційна система), віртуальні машини та віртуальні апаратні компоненти, такі як процесор, оперативна пам'ять, віртуальні жорсткі диски та віртуальні мережеві адаптери.

Основні функції Hyper-V включають створення та управління декількома віртуальними машинами, встановлення різних операційних систем, ефективний розподіл апаратних ресурсів між ними, ізоляцію середовищ та збереження стану віртуальної машини за допомогою контрольних точок.

 ## Відповіді на питання
 
   **1. Перерахуйте етапи для розгортання операційної системи на базі віртуальної машини VirtualBox.**
      
       Гіпервізори типу 1 (Bare Metal) встановлюються безпосередньо на фізичне обладнання. Вони мають прямий доступ до ресурсів («заліза»), що забезпечує найвищу швидкість роботи, стабільність та безпеку. Завдяки повній ізоляції віртуальних машин цей тип є стандартом для великих дата-центрів, серверних інфраструктур та хмарних сервісів.
       Гіпервізори типу 2 (Hosted) працюють як додатки всередині вже існуючої операційної системи (Windows, Linux тощо). Вони прості у використанні та не потребують складного налаштування, проте працюють повільніше, оскільки змушені запитувати ресурси через основну ОС. Це ідеальний вибір для персонального тестування програм, навчання та розробки на домашньому комп’ютері.
       Головна відмінність полягає в архітектурі: перший тип працює автономно на обладнанні для максимальної продуктивності, а другий — як програма в системі для зручності користувача. Перший тип обирають для бізнес-завдань, а другий — для індивідуальної роботи та експериментів.
  
  **2. Чи є якісь апаратні обмеження при встановленні 32- та 64-бітних ОС?**
     
      Обмеження між 32-бітними та 64-бітними системами зумовлені насамперед архітектурою процесора та здатністю адресувати пам'ять. Головний апаратний бар'єр 32-бітних ОС полягає в ліміті оперативної пам'яті: вони бачать лише до 4 ГБ RAM, тоді як 64-бітні системи підтримують терабайти даних. Що стосується процесорів, то 64-бітні CPU є універсальними та запускають обидва типи систем, тоді як застарілі 32-бітні процесори фізично не спроможні працювати з 64-бітним кодом. Для віртуалізації важливо, щоб фізичний процесор підтримував технології Intel VT-x або AMD-V, інакше ви не зможете запустити 64-бітну гостьову ОС навіть на потужному залізі. Також сучасні 64-бітні системи вимагають наявності інтерфейсу UEFI та специфічних інструкцій процесора, які відсутні у дуже старих моделях.
      
   **3. Які основні етапи при встановленні OS Linux в текстовому режимі?**

       Встановлення Linux у текстовому режимі (через інсталятори на кшталт ncurses) починається з вибору мови системи, розкладки клавіатури та налаштування мережевого з’єднання. Наступним критичним етапом є розмітка диска, де користувач вручну або автоматично створює розділи, визначає точки монтування (наприклад, / та swap) і обирає файлову систему. Після цього виконується створення облікового запису адміністратора (root) та звичайного користувача, вибір часового поясу, а також вибір програмних пакетів та графічного оточення, якщо вони потрібні. Фінальним кроком є безпосереднє копіювання файлів на диск, встановлення завантажувача (зазвичай GRUB) у головний завантажувальний запис та перезавантаження системи для входу у вже готове середовище.
      
   **4. Яким чином можна до установити графічні оболонки Gnome та KDE в Linux, якщо вона вже встановлена в текстовому режимі (вкажіть необхідні команди та пакети)?**

      Щоб встановити графічне середовище в уже розгорнутій системі Linux, необхідно скористатися стандартним менеджером пакетів (наприклад, apt для Debian/Ubuntu або dnf для Fedora) та встановити відповідні групи пакетів. Для встановлення GNOME зазвичай використовується команда sudo apt install ubuntu-desktop (або gnome-shell), а для KDE Plasma — sudo apt install kde-full (або kde-plasma-desktop). Разом із самими оболонками система автоматично завантажить дисплейний менеджер (GDM для GNOME або SDDM для KDE), який відповідає за графічне вікно входу. Після завершення завантаження та інсталяції пакетів необхідно перезавантажити комп'ютер або вручну запустити графічну службу командою sudo systemctl start gdm (чи sddm), після чого система автоматично перейде з текстової консолі до графічного інтерфейсу.
      
   **5. Дайте коротку характеристику графічних інтерфейсів, що використовуються в різних дистрибутивах Linux ( Варіант 9 - Xfce та Fvwm )**
    Xfce — це легке та стабільне робоче середовище, що базується на бібліотеці GTK і орієнтоване на високу продуктивність при низькому споживанні системних ресурсів. Воно пропонує класичний візуальний стиль із панеллю завдань та меню, що робить його ідеальним вибором для старих комп’ютерів або серверів, де потрібен графічний інтерфейс без зайвих візуальних ефектів.
   Fvwm (F Virtual Window Manager), своєю чергою, є надзвичайно потужним і гнучким віконним менеджером, який славиться своєю мінімалістичністю та майже безмежними можливостями кастомізації через конфігураційні файли. На відміну від сучасних інтегрованих середовищ, Fvwm споживає мізерну кількість оперативної пам'яті, але потребує від користувача навичок ручного налаштування, тому він популярний серед досвідчених користувачів та прихильників систем старого зразка.

## Контрольні запитання

**1. Порівняйте гіпервізори типу 1 та типу 2, яка між ними відмінність та сфера їх застосування?**
   Порівняння гіпервізорів: Гіпервізори типу 1 (Bare Metal) встановлюються прямо на апаратне забезпечення, забезпечуючи максимальну швидкість і безпеку, тому використовуються в серверних інфраструктурах і дата-центрах. Гіпервізори типу 2 (Hosted) працюють як програми всередині звичайної ОС, що зручно для навчання та тестування, але знижує продуктивність через додатковий рівень абстракції. Головна відмінність полягає в місці розміщення: на «залізі» або поверх іншої операційної системи.
   
**2. Розкрийте поняття «GNU GPL», яка його основна концепція?**
   Концепція GNU GPL: GNU General Public License — це ліцензія на вільне програмне забезпечення, основна концепція якої полягає у наданні користувачам чотирьох свобод: запускати, вивчати, поширювати та модифікувати програму. Вона використовує принцип «copyleft», який вимагає, щоб будь-які змінені або похідні версії програми також залишалися вільними та поширювалися на тих самих умовах.
   
**3. В чому суть програмного забезпечення з відкритим кодом?**
   Суть програмного забезпечення з відкритим кодом: Суть Open Source полягає у відкритому доступі до вихідного коду програми для будь-кого, що дозволяє спільноті перевіряти його на безпеку, виправляти помилки та додавати нові функції. Це стимулює інновації та прозорість, оскільки розвиток продукту не залежить від однієї компанії, а базується на колективній співпраці розробників з усього світу.
   
**4. Що таке дистрибутив?**
   Дистрибутив — це повноцінна операційна система, яка включає ядро Linux, системні бібліотеки, набір прикладних програм та систему управління пакетами, зібрані в єдиний інсталяційний образ. Кожен дистрибутив (як-от Ubuntu, Debian чи Kali) має свою специфіку, націлену на певні завдання: від домашнього використання до професійної кібербезпеки та адміністрування серверів.
   
**5. Які задачі системного адміністрування можна реалізувати на базі ОС Linux?**
     Задачі системного адміністрування в Linux: На базі Linux можна реалізувати повний спектр адміністративних завдань, включаючи керування користувачами та правами доступу, розгортання вебсерверів (Apache, Nginx), баз даних та поштових систем. Також Linux дозволяє налаштовувати мережеві екрани (firewalls), моніторити стан системи, автоматизувати рутинні процеси за допомогою скриптів (Bash, Python) та створювати надійні системи резервного копіювання.
    
**6. Як пов'язані між собою ОС Android та Linux?**
    Android побудований на базі модифікованого ядра Linux, яке відповідає за взаємодію з апаратним забезпеченням, керування пам'яттю та процесами. Хоча Android використовує ядро Linux як фундамент, він суттєво відрізняється від класичних дистрибутивів власною бібліотекою мови Java, графічним інтерфейсом та відсутністю стандартних утиліт GNU, що робить його спеціалізованою мобільною платформою.
    
**7. Основні можливості та сфера використання Embedded Linux?**
    Вбудований (Embedded) Linux — це оптимізовані версії ядра для роботи на пристроях з обмеженими ресурсами, як-от роутери, розумні гаджети, автомобільні системи та промислове обладнання. Його головними можливостями є гнучкість налаштування під конкретне залізо та низьке споживання енергії, що дозволяє створювати надійні вузькоспеціалізовані системи для IoT.
    
**8. Яким чином можна змінити типу завантаження Linux: в текстовому режимі (3 рівень) або графічному (рівень 5)? Чим відрізняються режими CLI та GUI?**
    Змінити тип завантаження в сучасних системах можна командою systemctl set-default multi-user.target для текстового режиму або graphical.target для графічного. Режим CLI інтерфейс командного рядка споживає мінімум ресурсів і керується текстом, тоді як GUI (графічний інтерфейс) пропонує візуальне середовище з вікнами та мишею, що зручніше для користувача, але вимагає значно більше пам'яті та потужності процесора.

### Висновок ( українська мова )

У результаті виконання лабораторної роботи було детально вивчено принципи функціонування віртуальних середовищ та особливості операційних систем родини Linux. Під час підготовчого етапу вдалося засвоїти базову термінологію та класифікацію гіпервізорів, виділивши ключові відмінності між автономними рішеннями типу Bare Metal та хостовими програмами для віртуалізації. На прикладі конкретних інструментів, таких як VirtualBox, було розглянуто процес розгортання віртуальних машин, що дало розуміння необхідності врахування апаратних обмежень процесора та оперативної пам'яті при роботі з різними архітектурами. Опрацювання матеріалів курсу «Linux Essentials» та перегляд демонстраційних відео дозволили порівняти методи встановлення систем через Dual Boot та WSL, а також навчитися керувати програмними пакетами й графічними оболонками.

### Висновок ( англійська мова )

As a result of this laboratory work, the principles of virtual environments and the specific features of Linux-based operating systems were studied in detail. During the preparatory stage, basic terminology and hypervisor classification were mastered, highlighting the key differences between Type 1 (Bare Metal) and Type 2 (Hosted) solutions. Using specific tools such as VirtualBox, the process of deploying virtual machines was examined, providing an understanding of hardware constraints regarding CPU and RAM when working with different architectures. Reviewing the "Linux Essentials" course materials and demonstration videos allowed for a comparison of installation methods like Dual Boot and WSL, as well as learning how to manage software packages and desktop environments.
Special attention was paid to the flexibility of Linux interfaces, particularly the comparison of Xfce and Fvwm, which demonstrated the advantages of open-source software in terms of customization and resource optimization. The answers to the control questions confirmed a solid understanding of free software philosophy, the GNU GPL license concept, and the role of the Linux kernel in modern ecosystems, including Android and embedded systems. The acquired skills in configuring runlevels and transitioning between CLI and GUI modes are fundamental for further studies in Cybersecurity, as they enable effective administration of server solutions and ensure stable operation of network nodes in a secure virtualized environment.
