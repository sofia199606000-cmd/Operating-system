### Словник основних англійських термінів

 **Command Line Interface (CLI)** — інтерфейс командного рядка, який забезпечує точний контроль, високу швидкість та можливість автоматизації завдань за допомогою скриптів.
 
 **Shell** — оболонка; командний інтерпретатор, який перекладає команди, введені користувачем, у дії, що виконуються операційною системою.
 
 **Bash (Bourne Again Shell)** — найбільш розповсюджена оболонка для дистрибутивів Linux.
 
 **Prompt** — рядок запиту (запрошення), який зазвичай містить інформацію про користувача та систему (наприклад, sysadmin@localhost:~$).
 
 **Command** — команда; програмне забезпечення, яке при виконанні в CLI здійснює певну дію на комп'ютері.
 
 **Options** — параметри (опції); використовуються для модифікації базової поведінки команди.
 
 **Arguments** — аргументи; використовуються для надання команді додаткової інформації, такої як ім'я файлу або користувача.
 
 **Case-sensitive** — чутливість до регістру; властивість Linux, згідно з якою команди, файли та змінні мають вводитися точно так, як вони написані (з урахуванням великих та малих літер).
 
 **Alias** — псевдонім; можливість створювати короткі назви для довгих команд.
 
 **Variable** — змінна; функція, що дозволяє користувачеві або оболонці тимчасово зберігати дані в пам'яті.
 
 **Quoting** — використання лапок; механізм, що вказує системі ігнорувати спеціальне значення символів або обробляти текст особливим чином.
 
 **Internal (Built-in) commands** — внутрішні команди; команди, що вбудовані безпосередньо в саму оболонку.
 
 **External commands** — зовнішні команди; команди, що зберігаються у файлах, які оболонка шукає в системі.
 
 **Control statements** — інструкції керування; дозволяють використовувати кілька команд одночасно або запускати додаткові команди (наприклад, ;, &&, ||).

## 4. Визначення ##
   
**Командний інтерпретатор** - це програма, яка приймає введені користувачем текстові команди та передає їх оболонці для подальшого виконання операційною системою.
**Оболонка** - це середовище командного рядка, яке виступає посередником між користувачем і ОС, перекладаючи текстові команди в конкретні дії системи.
**Команда** - це спеціальне програмне забезпечення, яке при запуску в інтерфейсі командного рядка (CLI) виконує певну визначену дію на комп'ютері.

## 5. Відповіді на питання ##
   
**- Яку базову інформацію надає рядок запрошення prompt?**

  Рядок запиту (prompt) є важливою частиною інтерфейсу, яка з'являється при запуску термінала та сигналізує про готовність оболонки до роботи. Типово він містить інформацію про поточного користувача (username), назву системи (hostname) та поточний робочий каталог, у якому перебуває користувач. Наприклад, символ тильди ~ у запиті використовується як скорочення для домашнього каталогу користувача.

**- Для чого команді потрібні параметри та аргументи?**

   Додаткове введення даних для команд необхідне для уточнення їхньої роботи та поділяється на два типи: опції (параметри) та аргументи. Опції використовуються для модифікації базової поведінки команди, тоді як аргументи надають додаткову інформацію, таку як назва файлу або ім’я користувача, над якими потрібно виконати дію. Кожен параметр чи аргумент зазвичай відокремлюється пробілом.
   
**- Яке призначення команд ls, які параметри та аргументи вона може мати? Наведіть 3 приклади.** 

   Команда ls призначена для виведення списку вмісту каталогів. Вона може приймати параметри для зміни формату виводу (наприклад, -l для довгого списку) та аргументи у вигляді шляхів до конкретних директорій. Приклади використання: по-перше, проста команда ls без аргументів виведе вміст поточної директорії; по-друге, ls -l відобразить файли з детальною інформацією про права доступу, розмір та дату; по-третє, ls /etc/ppp виведе вміст вказаного специфічного каталогу.
   
**- Яким чином можна використати історію команд, які переваги це надає?**  

   Історія команд (command line history) є однією з популярних функцій оболонки Bash. Вона дозволяє користувачеві переглядати та повторно використовувати раніше введені команди замість того, щоб набирати їх заново. Головними перевагами цього механізму є значне підвищення швидкості роботи в терміналі та можливість легко повторювати складні або довгі послідовності команд.

**- Яке призначення команди echo?** 

   Команда echo використовується в терміналі для виведення тексту або значень змінних на екран. Вона є незамінною при написанні скриптів та перевірці вмісту змінних, а також дозволяє комбінувати статичний текст із динамічними даними, що зберігаються в системі.
   
**- Охарактеризуйте поняття змінної в оболонці Bash, які типи змінних вона підтримує?**  

   Змінна - це функція, яка дозволяє користувачеві або самій оболонці тимчасово зберігати дані в пам'яті комп'ютера. Ці дані можуть використовуватися для надання критичної системної інформації або зміни поведінки Bash та інших команд. В оболонці Bash підтримуються два основних типи змінних: локальні (local) та змінні оточення (environment). 
   
**- Яке призначення команд env, export та unset?**  

  Ці команди призначені для керування змінними в системі. Команда env дозволяє переглянути поточні змінні оточення, export використовується для того, щоб зробити змінну доступною для дочірніх процесів (перетворити її на змінну оточення), а unset слугує для видалення змінної або функції з пам'яті оболонки.

**- Які команди для отримання довідки по командам в терміналі ви знаєте?**

  Основними інструментами для отримання допомоги є команда man (скорочення від manual), яка відкриває сторінки детальної документації для конкретної команди (наприклад, man uname). Також інформацію про тип команди можна отримати за допомогою команди type, а багато програм підтримують виклик вбудованої довідки через параметри, такі як -h або --human-readable у контексті відображення даних.

## Відповіді на контрольні запитання: ##

**1. Які типи команд існують в оболонці Bash?**

 В оболонці Bash виділяють кілька основних джерел та типів команд: внутрішні (built-in) команди, що вбудовані безпосередньо в оболонку; зовнішні команди, які зберігаються у вигляді окремих файлів у системі; псевдоніми (aliases), що дозволяють скорочувати довгі послідовності команд; та функції, які створюються користувачем для об'єднання кількох команд або перевизначення існуючих.
   
**2. Що таке змінні оточення? Які вони бувають. Як їх можна переглянути в терміналі?**

   Змінні оточення - це тип змінних у Bash, які зберігаються в пам'яті та використовуються для надання системної інформації або зміни поведінки оболонки та програм. Окрім змінних оточення, існують також локальні змінні. Переглянути повний список поточних змінних оточення можна в терміналі за допомогою спеціальної команди env.
   
**3. Опишіть змінну $PS1. Як в терміналі переглянути її вміст?**

Змінна $PS1 (Prompt String 1) визначає зовнішній вигляд та структуру основного рядка запиту (prompt), який користувач бачить перед введенням кожної команди. Вона містить спеціальні коди для відображення імені користувача, хоста та каталогу. Щоб переглянути її поточний вміст у терміналі, необхідно скористатися командою виводу echo $PS1.
   
**4. Як можна змінити значення змінної $PS1? Що при цьому відбудеться в рядку запрошенні в bash (рядок запрошення перед початком кожної команди). Як змінити значення цієї змінної не на поточний сеанс, а за замовчуванням?**

Значення $PS1 можна змінити шляхом прямого присвоєння нової константи в терміналі (наприклад, PS1="new_prompt> "), після чого вигляд рядка запиту миттєво зміниться на вказаний текст. Для того, щоб зміни стали постійними (за замовчуванням) і зберігалися після перезавантаження, необхідно внести відповідні налаштування у конфігураційні файли оболонки, такі як .bashrc або .bash_profile.

**5. Для чого використовують лапки в оболонці Bash? .**

Лапки використовуються для того, щоб вказати системі, що текст всередині них має оброблятися особливим чином або що спеціальні символи всередині мають бути ігноровані. Bash підтримує три типи лапок: подвійні ("), одинарні (') та зворотні (`), кожна з яких має свій ступінь впливу на інтерпретацію тексту оболонкою.
   
**6. Для чого використовують інструкції керування, які їх види Ви знаєте?**

 Інструкції керування (control statements) дозволяють користувачеві виконувати кілька команд одночасно або задавати умови для їхнього послідовного запуску. До основних видів інструкцій керування в Bash відносяться крапка з комою (;) для послідовного виконання, подвійний амперсанд (&&) для логічного "І" (виконання наступної команди лише за успіху попередньої) та подвійна вертикальна риска (||) для логічного "АБО".
    
**7. В чому різниця якщо в кінці рядку запрошення bash стоїть символ $ чи \#? Наприклад на екрані ми бачимо наступні записи**

 Ці символи в кінці рядка запиту вказують на рівень привілеїв поточного користувача в системі: символ $ означає, що сеанс запущено від імені звичайного користувача (наприклад, centos), тоді як символ # сигналізує про те, що користувач має права суперкористувача (root) і володіє повним доступом до керування системою.

**8. Яке призначення команд whereis та locate? Яка між ними відмінність?**

 Обидві команди використовуються для пошуку файлів, проте вони працюють по-різному: whereis фокусується на пошуку бінарних файлів, вихідних кодів та сторінок посібників для конкретних команд, тоді як locate здійснює швидкий пошук будь-яких файлів у всій системі, використовуючи попередньо проіндексовану базу даних (що швидше за прямий пошук, але потребує оновлення бази).

Особливу увагу було приділено роботі з командою ls для перегляду вмісту каталогів у різних форматах, зокрема з використанням параметра -l для довгого списку. Ми також навчилися створювати та використовувати змінні для зберігання даних, налаштовувати псевдоніми (aliases) для скорочення довгих команд та застосовувати різні типи лапок (подвійні, одинарні та зворотні) для коректної обробки тексту оболонкою. Крім того, ми відпрацювали використання інструкцій керування, таких як крапка з комою та логічні оператори, для виконання кількох команд одночасно, а також вивчили методи отримання допомоги в терміналі через Man-сторінки та спеціальні параметри команд.

 ## Висновок (англійською)

 During the laboratory work, we familiarized ourselves with the basic CLI mode commands in the Linux operating system and practiced working with text-based commands in the terminal. We studied the principles of the Bash command interpreter, which translates user commands into operating system actions, and learned to analyze the prompt that provides information about the user, system name, and current directory. In the course of the work, we mastered executing programs by typing their names, using options to modify core behaviors, and applying arguments to provide additional information such as filenames. Particular attention was paid to the ls command for listing directory contents in various formats, specifically using the -l option for long listings. We also learned to create and utilize variables for storing data, configure aliases to create short nicknames for longer commands, and apply different types of quoting—double, single, and back quotes—to control how the shell interprets text. Furthermore, we practiced using control statements, such as the semicolon and logical operators, to execute multiple commands at once and explored methods for obtaining terminal help via Man Pages and specialized command options.
